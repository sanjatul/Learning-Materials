# 🧠 Design Pattern vs Design Principle

---

# 🔷 What is a Design Pattern?

A **Design Pattern** is a reusable, proven solution to a common software design problem.

👉 It is a **template/structure** for solving recurring problems in software development.

### 💡 Key Idea:
- It tells you **how to solve a problem**
- It is **implementation-focused**

---

## 🧩 Examples of Design Patterns

### 1. Singleton Pattern
Ensures only one instance of a class exists.

### 2. Factory Pattern
Creates objects without exposing creation logic.

### 3. Repository Pattern
Separates data access logic from business logic (commonly used in ASP.NET Core).

### 4. Strategy Pattern
Allows switching algorithms at runtime.

---

# 🔶 What is a Design Principle?

A **Design Principle** is a general guideline or rule used to write clean, maintainable, and scalable software.

👉 It tells you **how to design good software**, not how to implement a solution.

### 💡 Key Idea:
- It tells you **what to follow**
- It is **guideline-focused**

---

## 📏 Examples of Design Principles

### 1. SOLID Principles
- **S**: Single Responsibility Principle  
- **O**: Open/Closed Principle  
- **L**: Liskov Substitution Principle  
- **I**: Interface Segregation Principle  
- **D**: Dependency Inversion Principle  

### 2. DRY Principle
Don’t Repeat Yourself.

### 3. KISS Principle
Keep It Simple, Stupid.

### 4. YAGNI Principle
You Aren’t Gonna Need It.

---

# ⚖️ Design Pattern vs Design Principle

| Feature | Design Pattern | Design Principle |
|--------|---------------|------------------|
| Type | Solution | Guideline |
| Nature | Concrete | Abstract |
| Focus | How to solve | How to design |
| Level | Implementation level | Design thinking level |
| Examples | Singleton, Factory, Repository | SOLID, DRY, KISS |

---

# 🧠 Simple Understanding

- **Design Principles = Rules for good design**
- **Design Patterns = Reusable solutions using those rules**

👉 Principles tell you *what to do*  
👉 Patterns tell you *how to do it*

---

# 🏠 Real-Life Analogy

### Building a house:

- **Design Principles** → Safety rules, strong structure, good ventilation
- **Design Patterns** → Kitchen layout design, staircase design, room arrangement

---

# 🚀 In Software Development (C# / .NET context)

- SOLID principles help you design clean architecture
- Design patterns help you implement solutions like:
  - Dependency Injection
  - Repository Pattern
  - Factory Pattern

---

# 🎯 Summary

- Design Principles = Guidelines for writing good software
- Design Patterns = Reusable solutions to software problems
